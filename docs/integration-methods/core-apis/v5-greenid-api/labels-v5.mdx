# Labels V5

:::note

**API Versions**

This documentation relates to Version 5, the current version of the greenID API. Version 5 became available to customers as of 07 Feb 2024.

:::

The `sourceField` and `rawData` elements of the responses contain labels and tooltips for the various data fields.  There are two forms these labels/tooltips can take: 

1. Handlebars templates of the form \{\{\{label\_name}}}.  For an example, see the `rawData` example in the [Interpreting Results](https://vixverify.atlassian.net/wiki/spaces/GREEN/pages/2310111315/Interpreting+Results+V5) section.  Handlebars templates allows a customer to customise the content of the label by using the Handlebars Javascript templating engine.  For more information on Handlebars, the reader is referred to the Handlebars website: [http://www.handlebarsjs.com/](http://www.handlebarsjs.com/)

2. A translated label using greenID defaults.  For an example, see the `sourceFields` example in the [Interpreting Results](https://vixverify.atlassian.net/wiki/spaces/GREEN/pages/2310111315/Interpreting+Results+V5) section above.

An account configuration option is set to determine which of these will be returned.  The default setting is to translate labels to the greenID defaults.  Please contact your GBG to change which option is used.

Alternatively, label translation can be disabled on a per-request basis by passing an extraData parameter to the [getFields](https://vixverify.atlassian.net/wiki/spaces/GREEN/pages/2310111283/Web+Method+Details+V5#WebMethodDetailsV5-getFields) web method.  If the `extraData` element contains a name-value pair with the name "disable.label.translation" and the value "true", then label translation will be skipped for that request.  The request below shows an example of a [getFields](https://vixverify.atlassian.net/wiki/spaces/GREEN/pages/2310111283/Web+Method+Details+V5#WebMethodDetailsV5-etFields) call with label translation disabled:
